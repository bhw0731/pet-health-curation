# 🎬 펫헬스 큐레이션 — 시연 시나리오 (심사 제출용)

> 정부 창업 사업 심사위원 대상 데모 가이드. 3~4분 분량의 시연 흐름과 발표 포인트를 정리했습니다.

---

## 1. 한 줄 소개
**"반려동물의 나이·종·고민만 입력하면, 커뮤니티 데이터 기반으로 맞춤 건강 가이드를 30초 만에 추천하는 서비스"**

## 2. 해결하는 문제 (Pain Point)
- 보호자는 반려동물 건강 정보를 **검색에 의존**하지만, 정보가 흩어져 있고 신뢰도가 낮음
- 우리 아이의 **나이·종·증상에 맞는** 정보를 골라내기 어려움
- → "검색 피로"를 줄이고, **개인화된 신뢰 가능한 가이드**를 제공

## 3. 시연 흐름 (Live Demo Flow)

| 단계 | 화면 | 발표 포인트 |
|---|---|---|
| ① 진입 | 랜딩 페이지 | "회원가입 없이 바로 체험 가능 — 진입 장벽을 최소화했습니다" |
| ② 가치 설명 | 서비스 소개 3카드 + 3단계 흐름 | "간편 입력 → 데이터 분석 → 맞춤 결과의 3단계 구조" |
| ③ 정보 입력 | 입력 폼 | 강아지 / 120개월(10살) / **관절·치아·체중** 선택 |
| ④ 유효성 검증 | (의도적으로) 고민 미선택 후 제출 | "입력 검증으로 데이터 품질을 보장 — 프론트+서버 이중 검증" |
| ⑤ 결과 확인 | 큐레이션 결과 | "노령기 + 관절을 **우선순위 상단**에 배치하는 규칙 기반 추천" |
| ⑥ 신뢰 요소 | 커뮤니티 데이터 예시 박스 | "향후 실제 커뮤니티 게시글·통계와 연동되는 지점" |
| ⑦ **차별화 포인트** | **같은 고민을 가진 보호자 후기** | "단순 정보 나열이 아니라, **같은 종·생애주기 보호자의 실제 경험**을 매칭해 보여주는 커뮤니티 기반 큐레이션" |

### 추천 시연 입력값
- **종**: 강아지
- **나이**: `120` (개월 = 10살, 노령기)
- **고민**: 관절 / 뼈, 치아 / 구강, 체중 / 비만

> 결과에서 **관절·치아가 체중보다 위에 정렬**되는 것을 보여주세요.
> → "나이 구간(노령기)과 고민을 조합해 가중치를 매기는 큐레이션 엔진"의 증거.

## 4. 기술적 강점 (질의응답 대비)
- **풀스택 동작하는 MVP**: React + Vite + Tailwind (프론트) / Node.js + Express (API)
- **이중 유효성 검증**: 클라이언트 1차 + 서버 2차 → 데이터 신뢰성
- **규칙 기반 큐레이션 엔진**: `나이 구간 × 고민` 조합 가중치 → 개인화
- **확장 설계**: 큐레이션 규칙을 데이터 레이어로 분리 → DB·AI 추천으로 고도화 용이

## 5. 향후 로드맵 (사업 확장성)
1. **데이터 고도화**: 정적 규칙 → DB(PostgreSQL) 이전, 실제 커뮤니티 데이터 수집
2. **추천 정교화**: 규칙 기반 → 커뮤니티 통계/ML 기반 개인화 추천
3. **커뮤니티 기능**: 보호자 Q&A, 후기, 전문가(수의사) 검수 콘텐츠
4. **수익화**: 맞춤 제품 추천(사료/영양제) 제휴, 병원 연계

## 6. 실행 방법 (현장 시연)
```bash
# 터미널 1
cd backend && npm install && npm run dev      # http://localhost:4000

# 터미널 2
cd frontend && npm install && npm run dev     # http://localhost:5173
```
→ 브라우저에서 `http://localhost:5173` 접속
