# 💰 펫헬스 큐레이션 — 수익 모델 (사업계획서용)

> 본 문서는 정부 창업 사업 제출용으로, 서비스의 시장·수익화 전략·단계별 로드맵·규제 대응을 정리합니다.
> 현재 MVP에는 **수익 전환 동선(제휴 상품 추천 + 클릭 추적)**이 목업으로 구현되어 있습니다.

---

## 1. 시장 규모 (TAM / SAM / SOM)

| 구분 | 정의 | 추정 |
|---|---|---|
| **TAM** (전체 시장) | 국내 반려동물 연관 산업 | 약 **8조 원**(2027년 약 15조 전망) |
| **SAM** (유효 시장) | 펫 헬스케어 + 펫 커머스(사료·영양제·용품) 온라인 | 약 **2~3조 원** |
| **SOM** (수익화 목표) | 3년 내 확보 가능한 건강 관심 보호자 트래픽 기반 | 연 **30~80억 원** 거래액의 제휴 수수료 |

- 국내 반려가구 **약 600만 가구**, 1인당 연간 양육비 평균 **150만 원+**
- 핵심 인사이트: 보호자는 **건강 정보 탐색 → 제품 구매**로 이어지는데, 그 사이가 파편화되어 있음. 우리는 **"맞춤 진단 → 맞춤 제품"**의 다리를 놓는다.

## 2. 핵심 가치 & 수익 연결 고리

```
[입력] 나이·종·고민
   → [분석] 건강 지수·맞춤 큐레이션  ← 사용자 가치(무료)
   → [전환] 고민에 맞는 제품 추천      ← 수익 발생 지점 ★
   → [유지] 실천 체크리스트·기록·재방문 ← LTV 증대
```

우리 서비스의 강점은 **"왜 이 제품인지"가 데이터로 설명된다**는 것. 단순 광고가 아니라 *"노령기 + 관절 고민이라 이 관절 영양제"*라는 맥락이 전환율을 높인다.

## 3. 수익화 모델 (3-Track)

### Track 1 — 제휴 커머스 (Affiliate) · *지금 MVP에 구현됨*
- 결과 화면의 **맞춤 제품 추천 → 제휴몰 구매 시 수수료**(통상 거래액의 3~10%)
- 현재 상태: 고민별 제품 데이터 파이프라인 + 클릭 추적 목업 완료 (`ProductRecommendations`, `lib/track.js`)
- 다음 단계: 쿠팡 파트너스·네이버 커넥트 등 제휴 API 연동, 전환 추적

### Track 2 — B2B 리드/광고
- **동물병원·펫보험·브랜드사**에 타겟 노출/리드 제공
- 예: "노령견 보호자 세그먼트"에 보험 가입 유도, 지역 병원 검진 예약 연계
- 필요: 일정 규모 트래픽 + 개인정보 동의 체계

### Track 3 — 프리미엄 구독 (장기)
- 수의사 검수 **심화 리포트**, 1:1 건강 상담, 반려동물별 건강 타임라인
- 월 **3,900~9,900원** 구독, 전환 가치가 검증된 뒤 도입

## 4. 단계별 로드맵

| 단계 | 기간 | 목표 | 주요 수익 |
|---|---|---|---|
| **Phase 1 — 검증** | ~6개월 | MAU 1만, 후기·데이터 축적 시작 | 제휴 커머스 |
| **Phase 2 — 성장** | 6~18개월 | MAU 10만, 제휴 API 정식 연동 | 제휴 + B2B 광고 |
| **Phase 3 — 확장** | 18개월~ | 수의사 콘텐츠·구독, 데이터 자산화 | 3-Track 전체 |

## 5. 핵심 지표 (KPI)
- **CTR**(추천 제품 클릭률), **CVR**(구매 전환율), **객단가**, **재방문율**, **저장(프로필)율**
- 현재 MVP는 `lib/track.js`로 클릭 이벤트를 적재 → 실서비스에서 전환 퍼널 분석의 기반

## 6. 리스크 & 규제 대응
- **의료/수의 정보 책임**: 모든 화면에 "참고용·수의사 상담 필요" 면책 고지 적용. 수의사 자문단 구성 예정 → 콘텐츠 신뢰도 확보
- **개인정보**: 현재는 로그인 없이 localStorage(기기 보관)로 최소 수집. 계정·B2B 단계에서 개인정보 동의·분리보관 체계 구축
- **광고 표시**: 제휴 영역에 **`AD·제휴` 명시** + 수수료 발생 고지(상생/표시광고법 대응) — MVP에 이미 반영
- **데이터 신뢰성**: 현재 커뮤니티 통계는 "예시" 라벨로 명확히 구분. 실데이터 축적 후 단계적 대체

## 7. 현재 MVP가 증명하는 것
- ✅ "진단 → 제품"의 **수익 전환 동선이 화면에서 동작**(클릭 추적 포함)
- ✅ 제품 추천이 **큐레이션 엔진과 같은 데이터 파이프라인**(고민 기반)에서 나옴 → 확장 시 그대로 실연동 가능
- ✅ 규제 대응(면책·광고표시)을 처음부터 UI에 반영

> 요약: **수익 구조의 "그림"과 "첫 동선"은 구현되어 있고, 남은 것은 트래픽 확보와 제휴/데이터의 실연동**입니다. 이는 본 사업 지원을 통해 Phase 1에서 검증할 핵심 가설입니다.
